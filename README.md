# Nutanix Enterprise AI Release Notes

A static website that publishes structured release notes for Nutanix Enterprise AI. When a new version ships, drop the PDF into `source_pdfs/`, run one command to convert it, and rebuild the site.

## How it works

1. **Convert** — a Python script extracts text from the PDF and sends it to the Claude API, which returns structured markdown with YAML frontmatter saved to `src/content/releases/`.
2. **Build** — Astro reads the content files and generates a static site with a version timeline, per-version detail pages, and full-text search via Pagefind.
3. **Deploy** — push to trigger a Cloudflare Pages / Netlify build.

## Site development

```bash
npm install
npm run dev        # local dev server at http://localhost:4321
npm run build      # production build + Pagefind index
npm run preview    # preview the production build locally
```

## Adding a new release

1. Drop the PDF into `source_pdfs/` — filename must contain the version slug, e.g. `Release-Notes-Nutanix-Enterprise-AI-v2_8.pdf`
2. Run the conversion script:
   ```bash
   python scripts/convert.py source_pdfs/Release-Notes-Nutanix-Enterprise-AI-v2_8.pdf
   ```
3. Review the generated file in `src/content/releases/v2_8.md` and edit if needed
4. Rebuild the site:
   ```bash
   npm run build
   ```

See [`scripts/README.md`](scripts/README.md) for setup instructions, API key configuration, and how to run the tests.

## Project structure

```
src/
  content/releases/   # one .md file per version (generated by convert.py)
  pages/              # index (timeline) and per-version detail pages
  components/         # VersionEntry, Search
  layouts/            # Base layout with dark theme
scripts/
  convert.py          # PDF → structured markdown via Claude API
  test_convert.py     # pytest tests for the conversion pipeline
  requirements.txt    # Python dependencies
source_pdfs/          # drop PDFs here (gitignored)
```
