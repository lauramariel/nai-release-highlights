summary: "Nutanix Enterprise AI 2.6 introduces the new AI Gateway for unified model routing and management, along with model fine-tuning support, speculative decoding, KV-Cache aware routing, and expanded foundational model support."
features:
  - title: "AI Gateway Mode via NAI UI"
    description: "A new AI Gateway mode provides a single, unified API to manage and route requests across all models hosted in the cloud, on-premises, or fine-tuned. Features include vendor choice across external providers (Anthropic, AWS Bedrock, NAI, OpenAI, GCP Vertex AI), cost control via granular rate limits per API key, reliability through configurable fallback models, and smart KV-Aware routing. Gateway mode can be enabled at install time; upgrades from prior versions only support Gateway Mode disabled."
    category: "Platform"
  - title: "Enhanced Navigation to Endpoints and API Client Keys"
    description: "When gateway mode is enabled, the left navigation pane provides access to Inference (Local Endpoints, Unified Endpoints, Providers) and API Client Keys (filterable by Endpoint Keys and MCP Client Keys tabs)."
    category: "Platform"
  - title: "Ability to Filter Audit Events by Support Telemetry"
    description: "In the Audit Events page, a new Support Telemetry checkbox filter allows users to filter audit events by support telemetry."
    category: "Platform"
  - title: "Enhanced Performance and Access for NAI Labs"
    description: "Users can now integrate their own external Milvus DB vector database instance into NAI Labs to improve performance and scalability. ML Users role can now access NAI Labs simultaneously. Newly available built-in safety models including LlamaGuard and NeMo Guard are available for experimentation. This is a tech preview feature."
    category: "MLOps"
  - title: "Object Detection and Safeguard Endpoints in Talk to My Data"
    description: "Users can now add an Object Detection Endpoint and a Safeguard Endpoint when configuring the Talk to My Data application in NAI Labs."
    category: "MLOps"
  - title: "Ability to Fine-Tune Models"
    description: "Users can fine-tune general-purpose models for company-specific domain language using the Low-Rank Adaptation (LoRA) technique, enabling model customization without large compute requirements. Customized models are automatically registered and managed by Nutanix AI for easy deployment. This is a tech preview feature."
    category: "MLOps"
  - title: "Enhanced Endpoint Creation"
    description: "Endpoint creation now includes advanced configuration options for validated and non-validated models, including enabling KV Cache aware routing (gateway mode required), selecting platform-provided engine arguments, enabling speculative decoding, selecting NAI-provided images as Engine Source, and configuring Engine Arguments for non-validated models."
    category: "Platform"
  - title: "Speculative Decoding"
    description: "Powered by vLLM's speculative decoding technique, this feature predicts multiple future tokens and verifies them with the main model to reduce inter-token latency and main model computation time, improving response performance for code generation, Chatbot, RAG, and summarization workloads."
    category: "Performance"
  - title: "Ability to Add Third-Party Credentials for Multiple Platforms"
    description: "Admins can add third-party credentials for multiple platforms. When gateway mode is enabled, credentials can be added for all platforms; when disabled, only Hugging Face and NVIDIA NGC are available. ML Users can only add credentials for Hugging Face and NVIDIA NGC."
    category: "Security"
  - title: "KV-Cache Aware Routing"
    description: "Intelligent routing powered by key-value (KV) cache awareness routes requests to GPUs that already have conversation history cached, reducing time-to-first-token for multi-turn chat and RAG workflows. Available only in AI Gateway mode. This is a tech preview feature."
    category: "Performance"
  - title: "Advanced vLLM Inference Sandbox"
    description: "Users can test and deploy custom vLLM versions with custom CLI arguments in an isolated environment, enabling access to the latest vLLM community versions and configurations without waiting for a quarterly NAI release."
    category: "MLOps"
  - title: "Ability to Add Remote MCP Server"
    description: "Remote MCP Server enables centralized and secure tool governance for AI agents, with RBAC-based access control, auditing, and simplified integration by aggregating multiple tool servers behind a single /mcp interface. Requires Gateway mode to be enabled. This is a tech preview feature."
    category: "Security"
  - title: "Support for New Models"
    description: "New AI LLMs are now supported, including allenai/Olmo-3-32B-Think, allenai/Olmo-3-7B-Instruct, allenai/Olmo-3-7B-Think, multiple mistralai/ministral-3 variants (3B, 8B, 14B Instruct and Reasoning), openai/gpt-oss-safeguard-120b, openai/gpt-oss-safeguard-20b (all via HuggingFace), and openai/whisper-large-v3 via NVIDIA NIM."
    category: "Core"
